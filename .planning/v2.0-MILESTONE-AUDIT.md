### Critical Integration Issue: DATA_DIR Import-Time Binding

**Root cause:** All 5 ingestor modules use `from pipeline.config import DATA_DIR` which binds `DATA_DIR` as a local name at import time. The conftest.py `isolate_data_dir` fixture patches `pipeline.config.DATA_DIR` (the module attribute), but the already-bound local names in ingestor modules are NOT affected.

**Impact on live tests:**
1. `fetch_and_save()` writes CSV to real `data/` directory (ingestor's local DATA_DIR)
2. Test asserts `pipeline.config.DATA_DIR / filename` exists (patched to tmp_path)
3. File is at `data/filename`, assertion checks `tmp_path/filename` → **AssertionError**

**Affected files:**
- `pipeline/ingest/abs_data.py` line 11: `from pipeline.config import ... DATA_DIR`
- `pipeline/ingest/rba_data.py` line 10: `from pipeline.config import DATA_DIR`
- `pipeline/ingest/asx_futures_scraper.py` line 22: `from pipeline.config import ... DATA_DIR`
- `pipeline/ingest/corelogic_scraper.py` line 18: `from pipeline.config import ... DATA_DIR`
- `pipeline/ingest/nab_scraper.py` line 20: `from pipeline.config import ... DATA_DIR`

**Fix options:**
- (a) Change ingestors to `import pipeline.config` and use `pipeline.config.DATA_DIR` at call time
- (b) Add per-module monkeypatches in live tests for each ingestor's DATA_DIR binding
- (c) Have live tests skip the isolate_data_dir fixture (check for live marker like block_network does)

Note: The conftest.py already documents this hazard (lines 42-45) for `SOURCE_METADATA` paths.

### Minor Integration Issue: Orphaned verify_summary.py

`scripts/verify_summary.py` was created in Phase 14. Its docstring claims `npm run verify` invokes it, but `package.json` does not include it. The `verify` script in Phase 15 was defined as:
```
"verify": "npm run test:fast && python -m pytest tests/python/ -m live -v && npx playwright test"
```
The original Phase 14 definition was `python pipeline/main.py && python scripts/verify_summary.py` but this was replaced, not merged.
