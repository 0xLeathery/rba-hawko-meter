### Strategy: Tier List Patching

Each test patches `pipeline.main.CRITICAL_SOURCES`, `pipeline.main.IMPORTANT_SOURCES`, and/or `pipeline.main.OPTIONAL_SOURCES` with minimal single-entry lists containing controlled mocks.

**Lambda detection:** `main.py` uses a lambda detection check: `'<lambda>' in str(module)`. Use lambda mocks for sources that use lambdas (most of them), or just replace with plain callables for simplicity since we're replacing the entire list.

For simplicity, use `MagicMock()` callables (they are callable but not lambdas, so `module.fetch_and_save()` will be called). Alternatively, use `lambda: mock_fn()` to match the lambda pattern.

**Best approach:** Replace tier lists with `[('Name', mock_callable)]` where `mock_callable` is a lambda wrapping a MagicMock. This ensures the lambda detection branch is exercised.

```python
def _make_lambda_mock(return_value=None, side_effect=None):
    """Return a lambda wrapping a MagicMock for use in tier lists."""
    m = MagicMock(return_value=return_value, side_effect=side_effect)
    return lambda: m()
```

```python
def test_all_success(self, monkeypatch):
    """All critical, important, optional sources succeed; normalization succeeds."""
    critical_mock = _make_lambda_mock(return_value={"rows": 10})
    important_mock = _make_lambda_mock(return_value={"rows": 5})
    optional_mock = _make_lambda_mock(return_value={"rows": 3})

    mock_status = {
        "overall": {"hawk_score": 55.0, "zone_label": "Balanced"},
        "metadata": {"indicators_available": 6, "indicators_missing": []},
    }

    monkeypatch.setattr("pipeline.main.CRITICAL_SOURCES", [("Test Critical", critical_mock)])
    monkeypatch.setattr("pipeline.main.IMPORTANT_SOURCES", [("Test Important", important_mock)])
    monkeypatch.setattr("pipeline.main.OPTIONAL_SOURCES", [("Test Optional", optional_mock)])
    monkeypatch.setattr("pipeline.main.NORMALIZATION_AVAILABLE", True)
    monkeypatch.setattr("pipeline.main.generate_status", lambda: mock_status)

    from pipeline.main import run_pipeline
    results = run_pipeline()

    assert results["status"] == "success"
    assert results["critical"]["Test Critical"]["status"] == "success"
    assert results["important"]["Test Important"]["status"] == "success"
    assert results["optional"]["Test Optional"]["status"] == "success"
    assert results["normalization"]["status"] == "success"
    assert results["normalization"]["hawk_score"] == 55.0
```

## Step 3: Modify `lefthook.yml` — Wire Coverage Check into Pre-Push Hook

**Current `unit-tests` command:**
```yaml
unit-tests:
  run: pytest tests/python/ -m "not live"
  timeout: "30s"
  fail_text: "Push blocked: Unit tests failed. Run npm run test:fast to see details."
```

**New `unit-tests` command:**
```yaml
unit-tests:
  run: pytest tests/python/ -m "not live" && python scripts/check_coverage.py --min 85
  timeout: "45s"
  fail_text: "Push blocked: Unit tests or coverage check failed. Run npm run test:fast to see details."
```

**Changes:**
1. Append `&& python scripts/check_coverage.py --min 85` to the `run:` value
2. Bump `timeout:` from `"30s"` to `"45s"` (coverage script adds ~2-3s overhead)
3. Update `fail_text:` to mention coverage check

**Rationale:** Coverage check reads `.coverage.json` written by pytest. Chaining in the same `run:` command ensures sequential execution (pytest must finish before coverage check reads its output). The parallel structure for lint vs. tests is preserved.
